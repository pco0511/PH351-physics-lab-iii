# Hall Effect Experiment – Data Description

## Experiment Overview

Hall effect measurements on Germanium samples (intrinsic, n-type, p-type).
An electromagnet coil produces a transverse magnetic field $B$; a constant
current $I$ is driven through the sample, and both the longitudinal voltage
(magnetoresistance) and the transverse (Hall) voltage are recorded.

---

## Raw Data (`data/raw/`)

### `current-field.xlsx`

Excel file mapping the **coil drive current** (A) to the resulting
**magnetic field** $B$ (mT) measured at the sample position.
Two sheets capture both sweep directions to characterise hysteresis.

| Sheet | Description |
|---|---|
| `increasing` | $B$ measured while increasing coil current |
| `decreasing` | $B$ measured while decreasing coil current |

Each sheet has a header row followed by data:

| Column | Variable | Unit |
|---|---|---|
| 1 | `I(A)` | A |
| 2 | `B(mT)` | mT |

22 data points per sheet: 1.0 A to 5.2 A in 0.2 A steps.

---

### `longitudinal/` and `transverse/`

Tab-separated time-series files exported from the measurement software.
Each sub-directory corresponds to one Ge sample type:

| Directory | Sample | Doping |
|---|---|---|
| `Ge/`   | Intrinsic germanium | undoped |
| `Ge-n/` | n-type germanium | electron-doped |
| `Ge-p/` | p-type germanium | hole-doped |

Within each sample directory the files are named by **coil current × 10**:

| Filename | Coil current |
|---|---|
| `10.txt` | 1.0 A |
| `12.txt` | 1.2 A |
| … | … |
| `52.txt` | 5.2 A |

> Longitudinal and transverse voltages are measured in **separate runs**
> because the instrumentation cannot record both simultaneously.

#### File Format

Lines 1–5 are a header; line 6 onward is tab-separated numeric data.

```
MIN=<v>   <v>   <v>
MAX=<v>   <v>   <v>
SCALE=<v> <v>   <v>
DEC=<v>   <v>   <v>
DEF="Time" t / s   "Current" I_A1 / A   "Voltage" U_B1 / V
<t0>   <I0>   <V0>
<t1>   <I1>   <V1>
…
```

- **Rows 1–4** (`MIN`, `MAX`, `SCALE`, `DEC`): instrument range / display
  settings for each channel (tab-separated values, first token is `KEY=value`).
- **Row 5** (`DEF`): column definitions – `"Label" varname / unit` per channel.
  *Column order can differ between files*; always parse from this row.
- **Rows 6+**: measurement data at 1 ms intervals.

#### Column Variables

| Variable | Role |
|---|---|
| `t` | Elapsed time |
| `I_A1` | **Sample current** $I_s$ driven through the Ge bar |
| `U_B1` | Measured voltage (longitudinal *or* transverse depending on directory) |

#### Time-Series Structure

Each file captures a transient (current ramp-up) followed by a stable
measurement period.

## HDF5 Data (`data/hdf5/`)

Generated by `scripts/raw_to_hdf5.py`.

### `voltage_measurements.h5`

```
/                               (root)
├── longitudinal/               (group) longitudinal voltage runs
│   ├── Ge/                     (group) intrinsic Ge
│   │   ├── coil_10/            (group) coil current = 1.0 A
│   │   │   ├── time_s          (dataset, float64, unit: s)
│   │   │   ├── sample_current_A (dataset, float64, unit: A)
│   │   │   └── voltage_V       (dataset, float64, unit: V)
│   │   ├── coil_20/
│   │   └── …
│   ├── Ge-n/
│   │   └── coil_XX/  …
│   └── Ge-p/
│       └── coil_XX/  …
└── transverse/                 (group) transverse (Hall) voltage runs
    ├── Ge/
    ├── Ge-n/
    └── Ge-p/
```

#### Group Attributes

| Level | Attribute | Description |
|---|---|---|
| root | `description`, `experiment`, `sample_material`, `source_directory` | global metadata |
| `longitudinal/` or `transverse/` | `description`, `voltage_type` | measurement type |
| sample group (`Ge`, `Ge-n`, `Ge-p`) | `description`, `doping` | sample info |
| coil group (`coil_XX`) | `coil_current_A` | coil current in A |
| | `source_file`, `source_path` | provenance |
| | `header_MIN_*`, `header_MAX_*`, etc. | raw instrument range per channel |

#### Datasets

All datasets within a `coil_XX` group are 1-D arrays of length $N$ (number of
time steps in that run), stored with gzip compression.

| Dataset | Unit | Description |
|---|---|---|
| `time_s` | s | Elapsed time |
| `sample_current_A` | A | Sample current $I_s$ |
| `voltage_V` | V | Longitudinal or transverse voltage $V$ |

---

### `calibration.h5`

```
/                               (root)
└── current_field/              (group)
    ├── increasing/             (group) sweep direction: increasing
    │   ├── coil_current_A      (dataset, float64, unit: A)
    │   └── B_field_mT          (dataset, float64, unit: mT)
    └── decreasing/             (group) sweep direction: decreasing
        ├── coil_current_A      (dataset, float64, unit: A)
        └── B_field_mT          (dataset, float64, unit: mT)
```

| Dataset | Unit | Description |
|---|---|---|
| `coil_current_A` | A | Coil drive current |
| `B_field_mT` | mT | Magnetic field at sample position |

## Conversion Script (`scripts/raw_to_hdf5.py`)

```bash
cd s2-hall-effect
python scripts/raw_to_hdf5.py
```

Dependencies: `h5py`, `numpy`.
For calibration: also `pandas` and `openpyxl` (or `xlrd` for `.xls`).

```
pip install h5py numpy pandas openpyxl
```

The script:
1. Parses the 5-line header of each txt file for column definitions and range metadata.
2. Identifies time, sample-current, and voltage columns by variable name (`t`, `I_*`, `U_*`).
3. Writes raw time series as compressed HDF5 datasets.
4. Reads the calibration Excel file (when present in `current-Bfield/`) and
   writes it to `calibration.h5`.
